# Mô tả: Bắt đầu sprint tiếp theo từ TASKS.md

Đọc `docs/TASKS.md` và xác định task tiếp theo chưa done.

Báo cáo:
1. Số task đã hoàn thành / tổng số
2. Sprint/task tiếp theo cần làm
3. File sẽ được tạo hoặc sửa
4. Lệnh verify sau khi xong

Sau khi tôi confirm, bắt đầu implement từng file một theo thứ tự dependency.
Dừng sau mỗi file để tôi verify trước khi tiếp tục.
